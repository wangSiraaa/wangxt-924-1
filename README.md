# 茶饮门店备料看板

面向店长、吧员和区域督导的门店备料管理看板，支持风险预警、日结管控和更正审计。

## 功能概览

### 店长工作台（📋 /manager）
- **原料库存录入**：录入所有原料当前库存，负数自动标红
- **今日销量预估**：按单品录入当日预估销量，已停售品自动禁用
- **安全库存设置**：设置每种原料的安全库存阈值
- **报损记录**：记录报损并跟踪处理状态
- **盘点管理**：标记盘点完成状态
- **更正记录**：日结后库存调整的唯一通道，保留审计痕迹
- **日结提交**：校验盘点/负库存/未处理报损，通过后锁定当日数据

### 吧员工作台（👨‍🍳 /staff）
- **待补料清单**：按分类显示缺货和低于安全库存的原料
  - 紧急缺货（红色）、低于安全库存（黄色）醒目提示
  - 显示替代原料及可用量
  - 关联受影响的可售单品
- **可售单品看板**：
  - 实时展示每款饮品可售状态（正常/原料不足/不可售/已停售）
  - 低于安全库存的原料自动高亮
  - 已停菜单品标记为灰色删除线

### 区域督导（📊 /supervisor）
- **多门店风险汇总**：高/中/低风险门店卡片
- **库存看板**：缺货总数、低于安全库存数量、盘点完成率、日结完成率
- **门店详情钻取**：点击门店查看完整缺货清单
- **盘点与日结完成度**：所有门店状态对比表

## 业务规则
1. 低于安全库存 → 醒目标记（黄/红底色+数字颜色）并自动关联影响可售单品
2. 已停菜单品 → 不进入销量预估录入，不生成备料任务
3. 日结拦截条件（任一即失败）：
   - 盘点未标记完成
   - 任一原料库存为负数
   - 存在未处理报损记录
4. 日结提交后：
   - 库存、预估、安全库存锁定不可直接修改
   - 库存调整只能走「更正记录」，原数不可覆盖
5. 所有数据保存于浏览器 `localStorage`

## 启动方式

```bash
# 1. 安装依赖
npm install

# 2. 启动开发服务器
npm run dev
# 默认 http://localhost:5173

# 3. 生产构建
npm run build

# 4. 预览构建结果
npm run preview

# 5. 运行 Smoke 测试
npm run smoke
```

## 数据存储说明

所有业务数据存储在浏览器 localStorage，键前缀 `tea_prep_`。
如需重置数据，在浏览器控制台执行：

```javascript
Object.keys(localStorage)
  .filter(k => k.startsWith('tea_prep_'))
  .forEach(k => localStorage.removeItem(k))
```
